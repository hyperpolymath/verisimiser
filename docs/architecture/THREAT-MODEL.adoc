= Threat Model: VeriSimiser
:toc:
:icons: font

== Overview

VeriSimiser augments existing databases with VeriSimDB octad capabilities.
It sits between applications and their databases, intercepting reads and writes
to maintain provenance chains, temporal versions, drift indices, and overlay
modalities.

== Trust Boundaries

=== Target Database

The target database is *not* under VeriSimiser's control. VeriSimiser observes
it but must never corrupt it.

* **Threat**: VeriSimiser bug writes to target database tables.
* **Mitigation**: Tier 1 sidecar isolation (proven in Idris2 ABI). Tier 1
  operations use read-only database connections. Zig FFI enforces write
  isolation at the C-ABI boundary.

=== Sidecar Storage

Provenance chains, temporal snapshots, and drift indices are stored in
external sidecars (SQLite files, VeriSimDB instances, or flat files).

* **Threat**: Sidecar tampering (modifying or deleting provenance records).
* **Mitigation**: SHA-256 hash chains are append-only. Each record hashes
  the previous record, making tampering detectable. Chain verification is
  available via `verisimiser provenance --verify`.

=== Interception Layer

VeriSimiser intercepts database events via CDC, triggers, or hooks.

* **Threat**: Interception misses events, leading to incomplete provenance.
* **Mitigation**: Backend-specific strategies chosen for reliability.
  PostgreSQL logical replication is the gold standard. Application-level
  hooks are fallback only.

=== VCL-total Queries

Type-safe queries cross both Tier 1 and Tier 2 boundaries.

* **Threat**: Query injection via malformed VCL-total strings.
* **Mitigation**: VCL-total queries are parsed and type-checked before execution.
  TypedQLiser provides compile-time validation. No string interpolation into
  backend queries.

== Drift Detection Integrity

* **Threat**: False negatives in drift detection (inconsistencies not flagged).
* **Mitigation**: 8-category drift classification ensures comprehensive coverage.
  Idris2 ABI proofs verify that all modalities are checked.

== Data at Rest

* **Threat**: Sensitive data in sidecar storage (provenance snapshots may contain
  entity state).
* **Mitigation**: Sidecar encryption is configurable. Temporal snapshots respect
  retention policies and are auto-pruned.
