= VeriSimiser ABI / FFI
:toc: macro
:source-highlighter: rouge

How VeriSimiser exposes its octad-augmentation core across a C ABI: an Idris2
ABI carrying machine-checked proofs, a Zig FFI implementing the C-compatible
surface, and the contract between them. Everything described here is verified
in CI by `.github/workflows/provable.yml`; this document tracks what is
actually in the tree, not an aspiration.

toc::[]

== Layout

[cols="1,3"]
|===
| Path | Role

| `src/interface/abi/Verisimiser/ABI/Types.idr`
| Core ABI types (octad dimensions, backends, result codes) and their proofs.

| `src/interface/abi/Verisimiser/ABI/Layout.idr`
| C-struct memory-layout proofs (size, alignment, per-field offset).

| `src/interface/abi/Verisimiser/ABI/Foreign.idr`
| `%foreign` declarations + safe Idris wrappers for every C symbol.

| `src/interface/ffi/src/main.zig`
| The Zig implementation of the C-ABI surface (`export fn verisimiser_*`).

| `src/interface/ffi/test/integration_test.zig`
| Tests that call the exported symbols through the compiled shared library.

| `src/interface/ffi/build.zig`
| Build: shared + static lib, `zig build test` (unit + integration).
|===

NOTE: The module path mirrors the module name (`Verisimiser.ABI.*`), so the
`.idr` files live under `Verisimiser/ABI/`. Run `idris2` from
`src/interface/abi` so the import root resolves.

== What is actually proven

The Idris2 ABI is not decorative ã these are total, hole-free proofs that
`idris2 --check` verifies on every CI run. The load-bearing ones:

[cols="1,2"]
|===
| Proof (module) | Statement

| `octadIsEight` (`Types`)
| The canonical dimension list has exactly 8 entries ã the "octad" invariant,
  by `Refl`.

| Non-null `Handle` / `DbConnection` (`Types`)
| A handle can only be constructed from a non-zero pointer
  (`{auto 0 nonNull : So (ptr /= 0)}`); `createHandle` / `createDbConnection`
  use `choose` to discharge it, so a null pointer yields `Nothing`, never a
  bogus handle.

| `tier1NeverWritesTarget` (`Types`)
| Tier-1 sidecar isolation: a Tier-1 dimension's storage is the sidecar, never
  the target table ã encoded so a counterexample does not type-check.

| `octadRecordValid`, `provenanceEntryValid`, `driftMeasurementValid`,
  `temporalSnapshotValid` (`Layout`)
| Each concrete C-struct layout is `CABICompliant`: its declared size is a
  multiple of its alignment, it is at least as large as the sum of its fields
  (`StructLayout`'s `aligned` / `sizeCorrect` invariants), and every field sits
  at an offset that is a multiple of that field's alignment (`FieldsAligned`).

| `alignUpCorrect` (`Layout`)
| The generic rounding lemma: `alignUp size align` is always a multiple of
  `align`. `alignUp` is *defined* as `ceilDiv size align * align`, so this is
  unconditional (it even holds for the degenerate `align = 0`).
|===

What is deliberately *not* claimed: there is no generic `checkCABI` or
`offsetInBounds` over arbitrary layouts/fields ã the latter is simply false for
an arbitrary field, and the former needs a per-field decision procedure. The
honest, machine-checked artefacts are the concrete `*Valid` instances above.
See the NOTE comments in `Layout.idr` for the reasoning.

== The C-ABI surface

Every symbol is `export fn` in `main.zig` and declared in `Foreign.idr`.
Handles are opaque: C callers only ever hold a `void*`. Result codes
(`verisimiser_*` returning `int`) follow the `Result` enum:

[cols="1,1,4"]
|===
| Code | Name | Meaning
| 0 | `ok` | success
| 1 | `error` | generic failure
| 2 | `invalid_param` | argument out of range (e.g. unknown dimension/operation)
| 3 | `out_of_memory` | allocation failed
| 4 | `null_pointer` | a required pointer argument was null
| 5 | `connection_failed` | target database connection failed
| 6 | `chain_corrupted` | provenance hash-chain integrity violation
| 7 | `sidecar_unavailable` | sidecar storage unavailable
|===

Function groups (see `Foreign.idr` for exact signatures): lifecycle
(`init`, `free`), connection (`connect`, `disconnect`), octad overlay
(`enable_dimension`, `get_active_dimensions`), Tier-1 provenance
(`record_provenance`, `verify_provenance`, `provenance_length`), Tier-1
temporal (`record_version`, `query_at_time`, `current_version`), Tier-1 drift
(`measure_drift`, `drift_score`, `drift_category_score`), VCL-total queries
(`vql_query`, `vql_free_result`), strings (`get_string`, `free_string`),
errors (`last_error`), version (`version`, `build_info`), and utilities
(`is_initialized`, `backend_supported`).

NOTE: A C header is not auto-generated today. `Foreign.idr` and `main.zig` are
the authoritative signatures; bind against them directly (examples below). A
hand-written `include/verisimiser.h` is tracked as future work ã until it
exists, the `build.zig` does not try to install one.

== Building and testing

[source,sh]
----
# Idris2 ABI ã type-check the proofs (run from the ABI root)
cd src/interface/abi
idris2 --check Verisimiser/ABI/Types.idr
idris2 --check Verisimiser/ABI/Layout.idr
idris2 --check Verisimiser/ABI/Foreign.idr

# Zig FFI ã build the libraries and run all tests
cd src/interface/ffi
zig build                 # shared + static lib into zig-out/lib
zig build test            # 8 unit tests + 35 C-ABI integration tests
zig build test-unit       # in-module unit tests only
zig build test-integration  # C-ABI integration tests only

# Cross-compile the FFI to another target
zig build -Dtarget=aarch64-linux
----

== Usage

=== From C

[source,c]
----
// Signatures mirror Foreign.idr / main.zig (no generated header yet ã declare
// the symbols you call, or write a thin verisimiser.h of your own).
#include <stdint.h>
#include <stdio.h>

extern void *verisimiser_init(void);
extern void  verisimiser_free(void *handle);
extern int   verisimiser_enable_dimension(void *handle, uint64_t entity, uint32_t dim);
extern int   verisimiser_record_provenance(void *handle, uint64_t entity,
                                           uint32_t op, uint64_t actor);
extern const char *verisimiser_last_error(void);

int main(void) {
    void *h = verisimiser_init();
    if (!h) return 1;

    // Enable the Provenance dimension (2) for entity 42, then record a create (0).
    if (verisimiser_enable_dimension(h, 42, 2) != 0 ||
        verisimiser_record_provenance(h, 42, 0, 0) != 0) {
        fprintf(stderr, "error: %s\n", verisimiser_last_error());
    }

    verisimiser_free(h);
    return 0;
}
----

Compile with `cc example.c -lverisimiser -L src/interface/ffi/zig-out/lib`.

=== From Idris2 (the safe wrappers)

[source,idris]
----
import Verisimiser.ABI.Foreign

main : IO ()
main = do
  Just h <- init
    | Nothing => putStrLn "init failed"
  Right () <- enableDimension h 42 Provenance
    | Left err => putStrLn (errorDescription err)
  free h
----

=== From Rust

[source,rust]
----
#[link(name = "verisimiser")]
unsafe extern "C" {
    fn verisimiser_init() -> *mut core::ffi::c_void;
    fn verisimiser_free(handle: *mut core::ffi::c_void);
    fn verisimiser_backend_supported(backend_id: u32) -> u32;
}

fn main() {
    unsafe {
        let h = verisimiser_init();
        assert!(!h.is_null());
        assert_eq!(verisimiser_backend_supported(1), 1); // sqlite
        verisimiser_free(h);
    }
}
----

=== From Julia / Deno

The same C symbols are callable via Julia `ccall` or a Deno FFI
`Deno.dlopen` over `libverisimiser`; the argument/return types are the C
types in the table above (`void*` handles, `uint32`/`uint64`/`int`/`double`).

== Contributing ã order of changes

When you touch the ABI/FFI, keep the layers in lock-step:

. *Idris2 ABI first* (`src/interface/abi/Verisimiser/ABI/*.idr`): change the
  types/layouts, update the affected proofs, and re-run `idris2 --check` until
  clean. If you add a struct, add its `StructLayout` witnesses and a
  `*Valid : CABICompliant` proof.
. *Zig FFI* (`src/interface/ffi/src/main.zig`): match the new types exactly;
  keep enum discriminants identical to the Idris `Result` / `OctadDimension` /
  `DatabaseBackend` values.
. *Tests*: extend the in-module unit tests and the C-ABI integration tests.
. *Verify*: `zig build test` and the three `idris2 --check`s must pass ã the
  `provable.yml` workflow runs exactly these.
